Assignment 1: House Price Prediction with Tabular Data

Resources:
  • Dataset: Melbourne Housing Snapshot (Kaggle)
  • Jupyter Notebook: Open in Google Colab

1. Abstract

This project tackles the problem of house price prediction using the Melbourne Housing dataset. The objective is to compare traditional machine learning approaches with deep learning for regression tasks on structured tabular data. Key aspects include:

  • Feature Engineering: Handling missing values, encoding categorical variables, and scaling numerical features.
  • Dimensionality Reduction: Using PCA to reduce feature space while retaining variance.
  • Model Comparison: Linear Regression, SVR, Random Forest, and Multi-Layer Perceptron (MLP).

2. Dataset Overview & Problem Statement

The task is a regression problem where the goal is to predict house prices based on various property attributes.

  • Target Variable: Price (log-transformed to handle skewness)
  • Features:
    • Numerical: Rooms, Distance, Bedroom2, Bathroom, Car, Landsize, BuildingArea, YearBuilt, Latitude, Longitude, Propertycount
    • Categorical: Suburb, Type, Method, CouncilArea, Regionname
  • Data Split: 80% Training / 20% Test

3. Exploratory Data Analysis (EDA)

Price Distribution

The house price distribution is right-skewed with presence of high-value outliers. Log transformation was applied to normalize the distribution for better model performance.

Missing Values

Several features had missing values, particularly BuildingArea (47%) and CouncilArea (12%). These were handled through imputation strategies.

Regional Analysis

House prices vary significantly by region, with Southern Metropolitan commanding the highest average prices.

Distance vs Price

A clear negative correlation exists between distance from CBD and house price, showing proximity value.

Correlation Analysis

Strong positive correlations with price were found for: Rooms (0.53), Bathroom (0.50), and Car spaces (0.42).

4. Data Preprocessing

A comprehensive preprocessing pipeline was implemented:

Feature Engineering Pipeline

  • Target Transformation: Log-transform of Price to handle skewness
  • Numerical Features: Mean imputation + Scaling (MinMaxScaler or StandardScaler)
  • Categorical Features: Most-frequent imputation + One-Hot Encoding
  • Feature Reduction: PCA with 90% or 95% variance retention

Example: Standard Scaler + PCA(0.95) reduced features from ~600 to ~150 dimensions

5. Model Implementation

Traditional Machine Learning Models

Three classical ML algorithms were tested with GridSearchCV (3-fold CV):

  • Linear Regression: Baseline linear model (no hyperparameters)
  • Support Vector Regression (SVR): Tuned C=[0.1, 1, 10], kernel=['linear', 'rbf']
  • Random Forest: Tuned n_estimators=[50, 100], max_depth=[10, 20]

Each model was trained with different combinations of scaler (MinMax/Standard) and PCA variance (90%/95%).

Deep Learning Model

A Multi-Layer Perceptron (MLP) was implemented for comparison:

  • Architecture: 5 hidden layers (256→128→64→32→16 neurons)
  • Activation: ReLU
  • Optimizer: Adam (learning_rate=0.001)
  • Training: 2000 max iterations with early stopping
  • Target Scaling: StandardScaler on target variable for stability

6. Results & Comparison

Scaler PCA Model Best Params MSE R² Score
Standard None MLP 5 layers (256→16) 0.0399 0.8539
Standard 0.90 SVR C=1, kernel=rbf 0.0413 0.8487
Standard 0.95 SVR C=10, kernel=rbf 0.0420 0.8462
MinMax 0.90 SVR C=1, kernel=rbf 0.0524 0.8083
Standard 0.90 Random Forest depth=20, n=100 0.0538 0.8033
Standard 0.95 Linear Regression — 0.0806 0.7052

Performance Analysis

  • Best Performer: SVR with Standard Scaler + PCA(0.90) achieved the highest R² score of 0.8487, explaining ~85% of price variance.
  • Scaler Impact: StandardScaler consistently outperformed MinMaxScaler across all models, particularly for SVR.
  • PCA Trade-off: PCA(0.90) vs PCA(0.95) showed minimal performance difference, but 0.90 offers better computational efficiency with fewer features.
  • Random Forest: Performed competitively (R²=0.803) and is more interpretable than SVR.
  • Deep Learning: MLP showed competitive performance but required more computational resources and careful tuning.
  • Linear Regression: Provided a reasonable baseline (R²=0.705) but limited by linear assumptions.

Key Insights

For tabular data regression: Traditional ML methods (especially SVR with RBF kernel) can match or exceed deep learning performance while being more efficient and easier to tune. Feature engineering and preprocessing (scaling, PCA) have a greater impact on model performance than the choice between traditional ML and deep learning.

7. Geospatial Analysis

An interactive heatmap visualization was created using Folium to show the geographic distribution of house prices across Melbourne. Red areas indicate higher prices, while blue areas show lower prices.

8. Conclusion

This project demonstrated effective house price prediction using machine learning on tabular data. Key takeaways:

  • SVR Dominance: Support Vector Regression with RBF kernel achieved the best performance (R²=0.849), proving that kernel methods excel at capturing non-linear relationships in housing data.
  • Preprocessing Matters: StandardScaler + PCA(0.90) provided the optimal balance between feature quality and dimensionality reduction.
  • Traditional ML Advantage: For structured tabular data, well-tuned traditional ML models can match deep learning while being faster to train and easier to interpret.
  • Feature Importance: Geographic features (Distance, Region) and property attributes (Rooms, Bathrooms) were the strongest predictors of price.
  • Practical Application: The final SVR model can be deployed for real-time price estimation with excellent accuracy and fast inference.

Assignment 2: Machine Learning with Text Data

Resources:
  • Dataset: AG News Classification Dataset (Kaggle)
  • Full Report: View PDF on Google Drive
  • Jupyter Notebook: Open in Google Colab

1. Abstract

This project addresses the problem of news topic classification using the AG News dataset. The primary objective is to implement, evaluate, and provide a comparative analysis of two different modeling paradigms:

  • Traditional Machine Learning: Utilizing TF-IDF feature extraction with algorithms like Naive Bayes, SVM, and Logistic Regression.
  • Deep Learning: Fine-tuning a pre-trained BERT (Bidirectional Encoder Representations from Transformers) model.

The study aims to quantify the trade-offs in performance, complexity, and computational cost between these approaches.

2. Dataset Overview & Problem Statement

The task is a multi-class classification problem where the goal is to categorize news articles into one of four categories based on their title and description.

  • Subset Used: 10,000 samples for training and 1,000 samples for testing.
  • Classes: World Sports Business Sci/Tech

3. Exploratory Data Analysis (EDA)

Before modeling, we analyzed the dataset structure to inform preprocessing decisions.

Class Distribution

The dataset is balanced. Each of the 4 categories contains an equal number of samples (~2,500 in training), which prevents model bias.

Text Length Analysis

The distribution of word counts is right-skewed. Most articles are of moderate length, result in the choice of max_len=128 for BERT tokenization.

Word Frequency

After removing stopwords, frequent terms included generic reporting words like "new", "said", "ap", "reuters".

4. Data Preprocessing

Different preprocessing pipelines were implemented for the traditional and deep learning approaches:

  • Traditional ML (TF-IDF):
    • Vectorization: The raw text was converted into numerical vectors using Scikit-Learn's TfidfVectorizer.
    • Stopword Removal: Configured with stop_words='english' to remove common terms with little semantic meaning.
    • Feature Storage: The generated feature matrices and labels were saved as NumPy arrays for reproducibility.
  • Deep Learning (BERT Tokenization):
    • Tokenizer: Utilized the BertTokenizer.
    • Encoding: Texts were converted into input_ids and attention_masks.
    • Constraints: Sequences were truncated or padded to a fixed maximum length of 128 tokens.

5. Model Implementation

Traditional Pipeline

GridSearchCV with 3-fold cross-validation was used to systematically tune hyperparameters for several algorithms:

  • Multinomial Naive Bayes: Tuned alpha.
  • Logistic Regression: Tuned C parameter and solvers (liblinear, saga).
  • Support Vector Classifier (SVC): Tuned C and kernel (linear, rbf).
  • Random Forest: Tuned n_estimators and max_depth.

Deep Learning Pipeline

We implemented Deep Learning by fine-tuning the BERT model:

  • Architecture: BertForSequenceClassification with a classification head for 4 classes.
  • Optimizer: AdamW, specifically designed for transformer models.
  • Scheduler: Linear learning rate scheduler with warmup to stabilize training.
  • Training: Fine-tuned for 2 epochs on a GPU.

6. Results & Comparison

Model Configuration Test Accuracy Macro F1
BERT (Fine-tuned) bert-base-uncased 91.3% 0.9133
Logistic Regression C=10, Liblinear 87.2% 0.8729
SVC C=10, RBF Kernel 87.1% 0.8712
Multinomial NB Alpha=0.1 86.2% 0.8625
Random Forest Default 82.5% 0.8252

Performance Analysis

  • Best Performer (BERT): The fine-tuned BERT model achieved the highest accuracy of 91.3%. This superior performance is attributed to BERT's ability to capture deep contextual relationships between words, unlike the "bag-of-words" assumption in TF-IDF.
  • Traditional Baseline: SVC and Logistic Regression performed competitively (~87%), proving that classical methods with proper tuning remain effective for text classification.
  • Worst Performer (Random Forest): The Random Forest classifier yielded the lowest accuracy at 82.5%. While generally robust, it lagged behind other methods in this high-dimensional sparse text data scenario.

5. Conclusion

This project successfully implemented and compared both traditional and modern deep learning methodologies for the task of news topic classification on the AG News dataset. The key findings are:

  • Superiority of BERT: With 91.3% accuracy, BERT significantly outperformed all traditional models (+4%). Its ability to understand context is a key advantage over TF-IDF's bag-of-words approach.
  • Strong Baselines: SVC and Logistic Regression proved to be highly effective baselines (~87% accuracy), showing that well-tuned classical models are still relevant for simpler tasks or resource-constrained environments.
  • Trade-off: The choice of model depends on the application. For more accuracy, BERT is recommended. For speed and low latency, SVC/Logistic Regression provides an excellent balance of performance and efficiency.

Assignment 3: Image Classification with Deep Learning

Resources:
  • Dataset: Mammals Image Classification Dataset (45 Animals)
  • Full Report: View PDF on Google Drive
  • Jupyter Notebook: Open in Google Colab

1. Abstract

This project addresses mammal species classification using the Mammals Image Classification Dataset. The primary objective is to implement, evaluate, and compare different feature extraction approaches:

  • Traditional Feature Engineering: Using HOG+PCA with classical ML algorithms (SVM, Random Forest, Logistic Regression).
  • Deep Learning Feature Extraction: Leveraging pre-trained models (VGG-16, ResNet-50, EfficientNet-B0) as fixed feature extractors.
  • Transfer Learning: Fine-tuning a pre-trained ResNet-50 model end-to-end.

The study aims to quantify the trade-offs in performance and efficiency between traditional computer vision and modern deep learning approaches.

2. Dataset Overview & Problem Statement

The task is a multi-class image classification problem where the goal is to categorize images of mammals into one of 45 distinct species.

  • Total Images: 13,751 images (224×224 pixels, JPG format)
  • Classes: 45 mammal species
  • Data Split: 60% Training (8,250) / 20% Validation (2,750) / 20% Test (2,751)

3. Exploratory Data Analysis (EDA)

Before modeling, we analyzed the dataset structure to inform preprocessing decisions.

Class Distribution

The dataset shows mild imbalance (215-356 images per class, ratio ~1:1.65), requiring stratified splitting for fair representation.

Brightness Analysis

Average brightness follows a normal distribution centered at 110-120 (0-255 scale), indicating well-balanced lighting conditions across the dataset.

4. Data Preprocessing

Two preprocessing pipelines were implemented:

  • Traditional Pipeline: Resize (224×224), ToTensor, RandomHorizontalFlip, RandomRotation (±5°), ColorJitter
  • Transfer Learning Pipeline: Traditional transforms + ImageNet Normalization (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

5. Feature Extraction & Model Implementation

Traditional Approach: HOG + PCA

  • HOG Configuration: 128×128 window, 8×8 cells, 9 orientation bins → 15,876 features
  • PCA Reduction: 15,876 → 2,295 dimensions (95% variance retained)
  • Classifiers: SVM, Random Forest, Logistic Regression

Deep Learning Approaches

Three pre-trained models were used as feature extractors, then combined with traditional classifiers:

  • VGG-16: 512-dimensional features
  • ResNet-50: 2,048-dimensional features (also fine-tuned end-to-end)
  • EfficientNet-B0: 1,280-dimensional features

All models were pre-trained on ImageNet and used as fixed feature extractors (frozen weights).

6. Results & Comparison

Performance Analysis

  • Best Performer: EfficientNet-B0 + Logistic Regression achieved 93.4% accuracy, demonstrating that modern architectures provide superior feature representations.
  • Traditional Failure: HOG+PCA completely failed (<15% accuracy), proving hand-crafted features are insufficient for complex real-world images.
  • Deep Learning Success: All pre-trained models (VGG, ResNet, EfficientNet) achieved >87% accuracy, showing the power of transfer learning.
  • Key Insight: Once high-quality deep features were extracted, classifier choice made minimal difference (all >87%), proving that feature quality is the determining factor, not the classification algorithm.

7. Conclusion

This project successfully compared traditional feature engineering against modern deep learning for image classification. The key findings are:

  • Deep Learning Dominates: Pre-trained models (EfficientNet-B0: 93.4%) vastly outperformed traditional HOG features (<15%), a difference of over 78 percentage points.
  • Feature Quality Matters Most: The "battle" was won at the feature extraction stage. Once deep features were obtained, all classifiers performed similarly (>87%).
  • Best Approach: EfficientNet-B0 as a feature extractor combined with Logistic Regression provides state-of-the-art accuracy with excellent efficiency.
  • Practical Recommendation: For similar tasks, use modern pre-trained models as fixed feature extractors—this offers the best balance of accuracy, speed, and simplicity.
